# babysit-dependabot

dependabotが作成した依存関係更新PRを確認し、安全にマージできるか検証する。

## 起動時の絶対遵守事項

- dependabotブランチに無関係な変更を追加しない
- 変更内容がPR目的（依存更新）と一致しない場合は、別ブランチを切って別PRとすること
- force-push禁止
- rebase禁止（コンフリクト解決は`git merge`を使用すること）

## フロー

### ステップ1: PR内容の確認
- `gh pr list --author "app/dependabot"`で対象PRを列挙
- 各PRの変更内容がロックファイルと依存宣言ファイルのみに閉じているか確認

### ステップ2: CI結果の確認
- `gh pr checks <PR番号>`で各チェックの通過状況を見る
- 失敗がある場合、ログを取得して原因を特定

### ステップ3: 破壊的変更の有無を確認
- メジャーバージョンアップの場合、CHANGELOGまたはリリースノートを確認
- 互換性リスクがある場合はユーザーに判断を仰ぐ

### ステップ4: マージ判断
- パッチ・マイナー更新でCIパス→マージ候補として提示
- メジャー更新→必ずユーザー承認を得る

### ステップ5: マージ実行
- ユーザー承認後に`gh pr merge <PR番号>`を実行
- マージ方法はリポジトリ設定に従う

## 除外条件

- CIが失敗しているPRは自動マージしない
- 依存関係以外のファイル変更を含むPRは対象外
- ドラフトPRは対象外

## レポート形式

各PRについて以下を報告:
- PR番号とタイトル
- 更新対象と新旧バージョン
- CI結果
- 判定（マージ可/要確認/要却下）
- 判定理由
