# 01. コンセプトとペルソナ

## ブランドポジション

Lazy Note は速報・SNS と隣り合わせの場ではなく、**机に置いておきたい雑誌** のようなブログを目指します。
更新頻度・流入規模を競うのではなく、**1 記事を腰を据えて読み返せる** ことに価値を置きます。

### ヘッドコピー

> **Notes that age well.**

### サブコピー

> 急がない記録、急がせない言葉。

### 3 キーワード

| キーワード   | 意味                                                                                                       |
| ------------ | ---------------------------------------------------------------------------------------------------------- |
| **Editorial**  | 雑誌編集的な余白・組版・見出しヒエラルキー。1 行の最大幅 36rem、行間 1.85 で「読み物」の佇まいに寄せる。 |
| **Calm**       | 一覧の情報密度を抑える。タイル数を絞り、status バッジは色相を持たない sumi 階調にする。                  |
| **Living**     | アクセントとしての Persimmon (柿) と Citrus (柑橘) で、季節感と温度を残す。grainy gradient で紙の質感。   |

## 思想

- 「いつでも読める」ことを前提にする (急かさない)
- 検索流入で 1 記事だけ読む読者にも、SNS から短文で流れてくる読者にも、**没入の損失を最小化** する
- ブログ運営者本人が「半年後に自分の記事を読み返したくなる」UI である
- アクセシビリティは飾りではなく、**読み物としての質を担保する基礎条件** とする (AAA 本文 7:1)

## 4 種の読書シーン

リニューアル時に検証すべきユースケースを 4 種に絞り込みました。各シーンが「どこに価値を置くか」を明確化し、UI 判断のスタックランキングに使います。

### 1. 通勤読者 (朝・昼・夕)

- 5〜15 分のスキマ時間に 1〜2 記事
- 端末: iPhone / Android、片手・縦持ち
- 重視: 本文の読みやすさ、フォントレンダ、スクロール感
- 嫌うもの: 過剰なローディング演出、視差スクロール、追従要素のジャンプ

### 2. 夜更けの内省読者

- 22:00 以降、ベッド・ソファ・暗い書斎
- 端末: iPad / 大画面スマホ、ダークモード
- 重視: ダーク時の眩しさ抑制、行間、句読点の打ち方
- 嫌うもの: 暖色光のダーク背景 (色温度ちぐはぐ)、青すぎるリンク

### 3. 検索流入読者

- Google から特定 1 記事に直接着地
- 重視: 「ここはどんなブログか」が 1 スクロールで分かること、TOC、関連リンク
- 嫌うもの: ヘッダの圧、いきなりのモーダル、「最初の記事から見る」しか入口がないナビ

### 4. SNS 流入読者

- X / Bluesky 等から、断片化された注意で着地
- 重視: OG 画像の引き、タイトル組版、Above the fold で本文に到達できる構造
- 嫌うもの: スプラッシュ、Hero の無意味な大画像、Cookie バナー的圧

## ペルソナ

### Persona A: マエハラ (32, ソフトウェアエンジニア)

- 通勤の山手線で技術記事を読む
- ダークモードを iOS 設定に合わせて自動切替
- 「文字組が雑だと萎える」と公言する読者
- 重視: コードブロックの可読性、Mono フォント、行間
- 本ブログでの主な動線: 検索流入 → 1 記事熟読 → サイト名で再訪 (たまに)

### Persona B: シノダ (28, ライター・編集者)

- 夜にベッドで読む。スクリーンタイム的に「眠れる」UI を求める
- 文章の体裁・句読点・余白に敏感
- 重視: 紙の編集物に近い佇まい、アクセシビリティ
- 本ブログでの主な動線: SNS 流入 → ホームを散策 → アーカイブ閲覧

### Persona C: 著者本人 (運営者)

- 半年〜1 年後の自分が読み返す
- 重視: 既存記事の互換性、書きやすい入稿フロー、状態管理 (draft / published / archived)
- 嫌うもの: 既存 16 記事を改変する破壊的変更、フロントマター強制移行

ペルソナ C を **互換性の番人** として、データモデル設計 (`06-data-model.md`) では「既存 `## 投稿日時` を絶対に廃止しない」ことを死守ポイントに置いています。
